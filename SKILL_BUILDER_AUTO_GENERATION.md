# Skill Builder Auto-Generation Hook

## Overview

The Skill Builder auto-generation hook automatically creates personalized learning plans when users complete assessments and score low in specific cognitive dimensions.

## How It Works

### 1. Assessment Completion

When a user completes any of the three core assessments (Kolb, Sternberg, or Dual-Process), the results are submitted to the `/assessment/submit` endpoint.

### 2. Dimension Mapping

The system maps assessment framework scores to skill dimensions:

#### Kolb Learning Styles → Skill Dimensions
- **Diverging** (feeling + watching) → `emotional_regulation` + `curiosity`
- **Assimilating** (watching + thinking) → `metacognition`
- **Converging** (thinking + doing) → `problem_solving`
- **Accommodating** (doing + feeling) → `curiosity` + `emotional_regulation`

#### Sternberg Thinking Styles → Skill Dimensions
- **Analytical** → `metacognition` + `problem_solving`
- **Creative** → `curiosity`
- **Practical** → `problem_solving`

#### Dual-Process Decision Making → Skill Dimensions
- **Reflective (System 2)** → `metacognition` + `problem_solving`
- **Intuitive (System 1)** → `emotional_regulation`

### 3. Weakness Detection

The system identifies the **weakest dimension** (lowest percentage score) for each user.

**Threshold**: A dimension is considered weak if it scores **< 25%** (below 1/4 of total points).

### 4. Plan Generation

If a weak dimension is detected AND the user doesn't already have an active plan for that dimension:

1. **7-day plan** is automatically created
2. Plan includes daily activities:
   - 1 Brain Gym game
   - 1 reflection prompt
   - 1 real-world challenge
3. Plan is stored with key pattern: `skillplan:{userId}:{planId}`
4. `sourceResultId` field links back to the triggering assessment

### 5. User Notification

The frontend displays a success toast notification:

```
🎯 We created a 7-day [Dimension] plan to help you grow in this area! Check your Skill Builder.
```

## Implementation Details

### Server-Side (`assessment-routes.tsx`)

**New Functions:**
- `mapAssessmentToSkillDimensions()` - Maps assessment results to skill dimension scores
- `autoGenerateSkillPlan()` - Creates plan if weakness threshold is met

**Modified Endpoint:**
- `POST /assessment/submit` - Now returns `autoGeneratedPlan` object if plan was created

**Response Structure:**
```typescript
{
  success: true,
  resultId: string,
  autoGeneratedPlan: {
    planId: string,
    dimensionId: string,
    message: string
  } | null
}
```

### Client-Side (`AssessmentTaking.tsx`)

**Modified Behavior:**
- Captures `autoGeneratedPlan` from submission response
- Displays extended toast notification if plan was created
- Links user to Skill Builder feature

## Example Flow

1. **User completes Kolb assessment**
   - Diverging: 15%
   - Assimilating: 35%
   - Converging: 30%
   - Accommodating: 20%

2. **System detects weak dimensions:**
   - `metacognition`: 35% (Assimilating)
   - `problem_solving`: 30% (Converging)
   - `curiosity`: 20% (max of Diverging/Accommodating)
   - `emotional_regulation`: 20% (max of Diverging/Accommodating)

3. **Weakest dimension identified:**
   - Both `curiosity` and `emotional_regulation` tied at 20%
   - System picks first one: `curiosity`

4. **Plan auto-generated:**
   - 7-day Curiosity skill plan created
   - User sees toast: "🎯 We created a 7-day Curiosity plan to help you grow in this area!"

## Business Logic

### When Plans Are NOT Created

- User scored **≥ 25%** in all dimensions (no significant weaknesses)
- User already has an **active plan** for the weakest dimension (prevents duplicates)
- Auto-generation fails (error is logged, but assessment submission still succeeds)

### Prevention of Duplicate Plans

The system checks for existing active plans before creating a new one:

```typescript
const hasActivePlan = existingPlans?.some((plan: any) =>
  plan.dimensionId === weakestDimension.dimensionId &&
  plan.status === 'active'
);
```

## Key Value Store Patterns

### Assessment Results
- Pattern: `result:{userId}:{assessmentType}`
- Example: `result:user-123:kolb`

### Skill Builder Plans
- Pattern: `skillplan:{userId}:{planId}`
- Example: `skillplan:user-123:plan-abc-def`

### Cross-Reference
Each auto-generated plan includes `sourceResultId` linking back to the triggering assessment:

```typescript
{
  sourceResultId: "result:user-123:kolb",
  // ... other plan fields
}
```

## Testing

### Manual Test Scenario

1. **Create low-scoring assessment:**
   - Take Kolb assessment
   - Answer mostly "Strongly Disagree" for Diverging questions
   - This should score Diverging < 25%

2. **Verify auto-generation:**
   - Check server logs for `[AutoPlan]` messages
   - Confirm toast notification appears
   - Navigate to Skill Builder
   - Verify new plan exists for the weak dimension

3. **Test duplicate prevention:**
   - Complete another low-scoring assessment
   - Confirm no duplicate plan is created (check logs)

## Monitoring

### Server Logs

Look for these log patterns:

```
[AutoPlan] Detected weak dimension for user {userId}: {dimensionId} ({score}%)
[AutoPlan] ✓ Created plan {planId} for {dimensionId} (user {userId})
[AutoPlan] User {userId} already has active plan for {dimensionId}, skipping
[AutoPlan] No weak dimensions for user {userId} (weakest: {dimensionId} at {score}%)
```

### Error Handling

Auto-generation failures are **non-blocking**:
- Errors are logged but assessment submission continues
- User receives their assessment results regardless
- Plan can be created manually via Skill Builder UI
